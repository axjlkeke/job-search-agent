# 求职决策模型（V1 历史基线，当前本地 V2）

状态：V1 质量修复已发布到 Mac mini 内部 3000；V2 推荐组合修正已在本地完成，尚未部署；公网 3001 未切换
当前本地版本：`2026-07-22.v2`
内部 3000 版本：`2026-07-22.v1`
第一版范围：央国企校招与实习

## 1. 数据职责

- 职达主站岗位接口是完整、持续更新的岗位真源；决策发生时按学生资料和审核关键词只读查询，不把职业情报库的68,353条迁移快照写成主站岗位总量。
- 独立职业情报库保存企业、学校、招聘批次和官方证据；学生选定目标后，只有A/B级已核验官方证据能改变硬门槛。
- 确定性决策模型负责预判、排序、目标组合和行动成本；Dify只解释结果、追问信息和陪伴执行。

## 2. 输入

- 学历、专业、毕业年份、地域偏好、每周可投入时间；
- 简历、网申、面试、项目、实习、竞赛六项准备状态；
- 主站最新岗位的岗位ID、单位、岗位、企业类型、招聘类型、地域、学历、专业、届别、起止时间、更新时间和来源；
- 选定目标后的官方证据决策。

姓名、学校全称、手机号、身份证号、简历原文不会进入市场报告或主站岗位查询。

## 3. 输出

1. 求职准备度：只表示材料与经历准备水平，不是同类排名；
2. 市场价值：用当前候选、企业、地域、更新时间和字段完整度表达机会覆盖，不生成薪资或录取概率；
3. 岗位预判：学历、专业、届别、截止时间逐项输出一致、相关、冲突或未知；
4. 目标组合：主攻、备选、提升后冲刺各最多3个，并说明每个岗位为什么进入该组；
5. 行动成本：能力缺口、预计投入时间和第一行动；
6. 官方核验：选定岗位后使用职业情报API重新判断当前批次资格；
7. 产品触发：已购服务优先使用，未购服务只在对应卡点成立后出现。

## 4. 强制边界

- 主站数据库零写入；
- 原始岗位字段不能直接生成“当前可投”；
- `unknown`不能升级为满足；
- 明确字段冲突显示高风险，但保留长期路径；
- 截止时间缺失不启动倒计时；
- 机会排序分只表示核验优先级，不是概率；
- 主攻、备选和提升后冲刺只能由确定性证据规则分配，不能由能力补齐工时或大模型自由改组；
- 大模型只负责抽取候选字段、解释证据、提出追问和陪伴执行，不能绕过届别、学历、时效、风险和官方证据门槛；
- 每个结论带数据时间、来源和模型版本；
- 当前只实现央国企路线，公务员和事业单位使用独立招录适配器，不复用企业招聘判断。

## 5. 验收样本

- 专科学生选择总部高学历岗位：显示高风险与长期路径；
- 2028届电气学生选择电力央企：返回主站最新候选并形成多企业组合；
- 标题包含“电气”但专业字段为计算机：专业冲突预警；
- 岗位无学历、届别或截止时间：保持待核验；
- 岗位已截止：不进入当前主攻；
- 简历、实习、面试均缺失：形成行动成本与卡点；
- 已购服务：优先进入直接使用，不重复营销；
- 主站或情报服务不可用：失败关闭，不返回硬编码岗位或资格。

- 已截止候选继续保留历史和长期路径，但不得进入当前主攻、稳妥或冲刺；主攻还明确排除高风险候选。全部截止时第一行动必须转向新批次、补录或替代岗位。
- 新增 8 个无 PII 合成样本和独立评测器，覆盖 6 个专业、4 个学历层级、3 个毕业年份和不同准备状态，并把上游 503 与模型硬失败分离。

## 9. 2026-07-22 本地 V2 推荐组合修正

- 候选检索先执行“专业 + 学历 + 届别”的严格资料查询，再补充相邻岗位方向；严格命中会优先进入详细候选池，不再只显示为一个孤立数量。
- 组合规则改为证据驱动：原始字段无冲突的岗位进入主攻；仍有少量关键信息待核验的岗位进入备选；只有存在可核验的企业难度或岗位专属能力差距证据时才进入提升后冲刺。没有证据时冲刺组保持为空。
- 明确高风险和已截止岗位退出当前执行组合；其余未入选岗位进入观察池，不用前三名、准备工时或卡片数量强制凑齐三组。
- 能力补齐工时和资格核验工时分开计算。简历、项目、实习等能力缺口用于生成行动计划，不能反向决定一个岗位属于主攻还是冲刺。
- 以本科、2028 届、电气工程及其自动化样本进行主站只读验证：本次详细去重候选 179 条，其中资料严格匹配 69 条；当前组合为主攻 1、备选 3、提升后冲刺 0，观察池 56，另排除高风险 92、已截止 27。该结果是 2026-07-22 的实时快照，会随主站岗位变化。
- 本地 V2 只改前端、服务端只读查询和确定性规则；未连接数据库、未执行 SQL、未写入或修改主站，未部署到内部 3000 或公网 3001。
