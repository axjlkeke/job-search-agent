# 决策质量灰度 V1

状态：规则修复、Mac mini 内部 3000 发布与 8 组真实只读灰度完成；公网 3001 未切换
日期：2026-07-22（Asia/Shanghai）
适用范围：已发布灰度基线 `2026-07-22.v1`；本地推荐组合修正版 `2026-07-22.v2` 尚未部署

## 1. 目标

这轮灰度不验证“页面能打开”，而是验证决策系统的三件核心事情：

1. 数据来源是否仍为主站公开只读岗位接口，且不回传学生个人信息；
2. 路线与目标组合是否引用真实候选，不把已截止或明确高风险岗位放进当前主攻；
3. 决策是否对不同专业、学历、毕业年份和准备状态保持一致、可解释、失败关闭。

## 2. 固定样本

`evals/decision-quality-cases.json` 保存 8 个纯合成样本，覆盖：

- 电气、酒店管理、计算机、会计、法学、旅游管理 6 个专业；
- 高职、专科、本科、硕士 4 个学历层级；
- 2026、2027、2028 三个毕业年份；
- 起步、部分准备、执行准备充分三类能力状态。

样本不包含姓名、学校、手机号、邮箱、身份证号、简历原文或真实用户标识。

## 3. 硬门禁

`scripts/evaluate-decision-quality.mjs` 对每个样本检查：

- HTTP 200，`source.queryMode=main-site-decision`；
- 模型版本、只读来源、非概率分数含义和无学生 PII 声明正确；
- 响应不回显档案，候选与评估 ID 一致且没有重复；
- 当前主攻/稳妥/冲刺组合只引用候选集；
- 已截止岗位不得进入任何当前执行组合；
- 已截止或明确高风险岗位不得进入主攻组合；
- 目标确认前不得触发产品营销，产品类别必须在受控集合中；
- 不同专业、学历和年份达到样本定义的最低候选、风险、方向与届别门槛；
- 数据源 503 与决策规则失败分开计数，禁止把基础设施故障写成模型失败。

Mac mini 发布脚本同步新增两个硬门禁：`expiredCandidatesExcludedFromActivePortfolio` 和 `primaryPortfolioContainsNoExpiredOrHighRiskCandidate`。

## 4. 本轮修复

- 主站字段中的 `26届`、`25届` 只在明确带“届”时解析为 2026、2025；普通人数如“招生计划26人”不会被误认为毕业年份。
- 已截止岗位继续保留在历史/长期路径中，但从当前主攻、稳妥和冲刺组合全部排除。
- 如果候选全部截止，第一行动改为核验新批次、补录或替代岗位，不再引导学生继续执行过期批次。

## 7. 本地 V2 新增硬门禁

- `portfolioAuthority=deterministic-evidence-rules`：组合归属只接受确定性证据规则；
- `aiRole=explain-extract-never-override-gates`：AI 可以抽取和解释，不能覆盖资格门槛；
- 严格资料匹配岗位必须先进入详细候选池，再进行风险、时效和字段冲突核验；
- 全局能力补齐工时不能作为主攻、备选或提升后冲刺的分组条件；
- 没有企业难度或岗位专属差距证据时，提升后冲刺允许为空；
- 高风险和已截止岗位不能进入任何当前执行组合，未入选但仍有效的岗位进入观察池；
- 每个入组岗位必须输出分组理由和下一项核验任务。
