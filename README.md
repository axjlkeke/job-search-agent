# 求职Agent

面向学生的央国企职业规划与求职行动平台。首版 Web 原型把 AI 对话、90 天计划、岗位匹配、简历诊断和模拟面试放进同一套求职工作台。

## 当前状态

当前仓库是可运行、可交互的前端原型：

- 已完成桌面端与移动端响应式界面
- 已完成 AI 对话、计划勾选、岗位收藏、简历清单与面试入口等交互演示
- 所有岗位、分数、来源和回答目前均为演示数据
- 尚未接入真实用户系统、知识库、Dify/Agent 平台或现有产品服务

页面已经明确标出“原型模式 · API 待接入”和“演示数据”，不会把模拟结果伪装成真实检索结论。

## 本地运行

需要 Node.js `>=22.13.0`。

```bash
npm install
npm run dev
```

默认访问：`http://localhost:3000`

## 后端接入边界

推荐由本项目的服务端接口统一代理下游能力，浏览器不能直接持有 Dify、模型或知识库密钥。

建议将能力拆成以下服务端工具：

1. `search_knowledge`：调用现有 RAG 检索 API，返回文档标题、发布日期、命中片段与来源链接。
2. `update_career_plan`：生成或调整阶段、周任务、截止时间和完成条件。
3. `match_jobs`：按学生档案和岗位要求返回匹配依据，不直接编造竞争概率。
4. `audit_resume`：提交简历并返回结构化问题与优化动作。
5. `start_mock_interview`：创建面试会话、追问与复盘结果。
6. `recommend_service`：推荐平台课程或服务时明确标注“平台提供 / 付费可选”。

环境变量示例见 `.env.example`。后续接入时应补齐身份认证、多用户会话隔离、限流、日志脱敏和写操作二次确认。

## 产品原则

- 事实、推断和建议分开呈现，事实必须附来源和更新时间。
- 资料不足时明确拒绝下结论，不使用“稳进、保录、内部名额”等承诺。
- 任务必须具体、可检查；投递、预约、购买等动作必须由用户确认。
- 本产品不是招聘单位官方平台，招聘条件以单位最新公告为准。
