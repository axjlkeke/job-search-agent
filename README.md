# 求职Agent

面向学生的央国企求职决策与行动平台。当前本地 Beta 已跑通“建档 → 选定岗位 → 生成策略网络 → 执行 7 天行动 → 在真实卡点出现服务入口”的主链路；AI 顾问只有在知识库、Dify 和会话保护全部配置后才会开放。

完整的产品路径、架构、接口、数据库隔离、部署边界、验收和修复流程见 [`PROJECT_DOCUMENTATION-AGENT.md`](./PROJECT_DOCUMENTATION-AGENT.md)。

正式网页入口：<https://agent.zhidasihai.cn>。该站点使用独立 GitHub 仓库与独立 Vercel 项目，不占用主站路由，也不在主站增加入口。

## 当前已完成

- 学生建档：记录学校、学历、专业、届别、目标城市、每周投入和六类能力现状。
- 真实在招岗位：服务端只读调用职达现有岗位接口，按专业、学历与届别筛选，并显示来源和更新时间。
- 多目标路径：最多同时选择 3 个岗位，形成一张“共同主干 + 企业分支”的求职策略网络。
- 硬门槛判断：学历、专业、届别和截止时间由确定性规则判断；资料缺失时标记“未知”，不编造匹配分数或录取概率。
- 官方证据决策：选定岗位后由服务端把最小化档案发送到独立只读职业情报服务；只有 A/B 级、已核验，且申请链接、证据链接、官方响应岗位 ID 与岗位名均对应同一目标的证据才能改变硬门槛，未覆盖、身份不一致或服务不可用时安全降级为“待核验”。2026-07-18 对现有79条官方岗位页的限速实时只读复核为79/79身份、状态和资格字段一致。本地待发布版本还会在每次岗位决策前实时核验受支持的 Hotjob 官方页；官网关闭、串岗、超时或失败时旧证据不能继续产生“可直接投递”结论。主 Beta 本地版已把结果分为“官网实时核验、官网本次无法核验、已核验快照”，并拒绝隐私字段或证据状态互相矛盾的响应。六文件内容哈希发布包、18081 候选烟测、显式批准和失败自动回滚也已准备完成，但尚未上传或执行；当前 `127.0.0.1:18080` 仍是旧服务。
- 7 天行动：把能力缺口拆成有完成标准的任务，勾选状态刷新后仍保留。
- 主站资料接力：Agent 侧已实现一次性码、state、PKCE、脱敏快照校验、加密 HttpOnly 会话、显式填表核对和断开；本地模拟闭环已通过，正式主站端点尚未部署。
- 跨设备路径状态：主站用专用 HMAC 密钥生成不可逆、受众隔离的匿名工作区标识；Agent 只在自己的私有目录保存最多 3 个公开岗位快照和任务勾选。个人档案仍留在浏览器，不写主站数据库，也不把匿名标识返回浏览器。双端本地闭环已通过，正式环境仍保持关闭。
- 产品触发：只读加载职达在线产品目录；接力快照确认的已有权益优先直接使用，其他产品只在简历、网申或面试等真实卡点作为可选入口。
- 安全 AI 适配层：先调用现有 RAG 接口检索，再按目标企业过滤后交给 Dify；无匹配资料、无有效引用或流程未完成时拒绝展示答案，通过时附可核验的官方原文片段。
- 本地真实知识库：SQLite 保存来源、原文和不可变版本，FTS5 提供可靠回退检索；Dify 索引未完成或不可用时不会冒充成功。
- 证据可用性门禁：页面按“可回答证据、发现索引、残缺页”分层；栏目列表和只有标题的残缺页可以保存以便后续抓取，但不得进入 SQLite/Dify 回答结果。Dify 片段必须能回查到本地当前版本、映射已完成且正文 hash 一致，否则丢弃并安全回退。
- 目标企业双层门禁：主项目和知识库服务都会按学生已经选定的企业过滤结果；支持“中广核/中国广核”“中石油/中国石油”等受控简称，但不会把共享集团前缀的兄弟企业当成同一目标，并显式区分“中国石油/中石油”与“中国石化/中石化”。明确目标没有证据时返回空结果，由上层拒答，不拿其他企业公告拼接。
- 公告变更安全闸门：同一公告的截止日、招聘对象、学历、专业要求、工作地点、招聘人数、官方投递入口、分学历年龄上限、分学历工作年限、英语四/六级及分数、届别、境内外毕业/认证日期、笔试/初选/测评/面试日期、投递次数或撤回/延期发生高置信变化时，候选版本先隔离并等待人工批准；报名区间兼容“完整起始年 + 月日终点”并保留 17:00 等具体时刻，只有能安全判断为同一年时才继承年份；高校毕业生、未就业毕业生、留学回国人员、社会人员和系统内外人员分别保留，学历、年龄、经验、外语和境内外时间要求也保留适用人群映射；考试安排会保留类型、暂定/确定状态和原文是否缺少年份，专业与明确标注的地点清单会忽略顺序、行政区后缀和分隔符差异；招聘人数只接受明确人数/名额字段或本次、计划、拟、公开招聘总量，报名人数、复审人数、岗位数量、开考比例和人月不混入；投递入口只接受报名/网申/投递标签或紧邻登录动作的网址、邮箱，普通公司官网、公告来源和客服邮箱不混入；企业介绍、待定字段和 OCR 技术补全也不会被误当成官方规则变化。
- 跨公告变更关系：官网用新 URL 发布更正、撤回、延期或部分岗位取消时，新公告与高置信旧公告先暂停检索；人工批准后只有明确整批终止会永久替代旧公告，其余变更进入待对账隔离，避免把仍有效岗位整体作废。
- 知识来源治理：已支持域名/路径白名单、受限批量同步、待审核项去重、覆盖率报告和 Mac mini 每日 04:10 同步任务。
- 本地进度页：开发期间可通过 `http://localhost:3000/progress` 查看当前里程碑。
- 新版工作台：可通过 `http://localhost:3000/v2` 单独查看 Studio 版产品界面；根路径旧版仍保留，便于对照和回退。

当前实现不会写入职达生产数据库，也不会自动投递、预约或购买。

## 本地运行

需要 Node.js `>=22.13.0`。

```bash
npm install
npm run dev
```

- 产品页：`http://localhost:3000`
- 新版工作台：`http://localhost:3000/v2`
- 进度页：`http://localhost:3000/progress`

`/v2` 是新版的独立观察入口；当前 `/` 旧版路由保留，不会因新版迭代而被覆盖。

学生档案继续保存在当前浏览器的 `localStorage` 中，键名为
`job-agent-workspace-v1`，30 天后自动失效。未连接主站时，已选岗位和任务
完成状态也只保存在本机；连接主站且服务端配置 `JOB_AGENT_WORKSPACE_DIR` 后，
这两类非个人数据会同步到 Agent 自己的匿名工作区，实现跨设备继续。清除按钮
会在连接状态下同时清除本机资料和 Agent 侧路径进度。

## 数据与接口边界

浏览器只访问本项目的服务端接口，不直接持有职达、知识库或 Dify 密钥。

| 本项目接口 | 当前作用 | 状态 |
| --- | --- | --- |
| `GET /api/jobs` | 只读检索职达在招岗位 | 已接通 |
| `POST /api/market-report` | 用最小档案生成真实岗位市场报告 | 已接通；只读岗位，不接收姓名和学校，同类排名无样本时拒绝计算 |
| `GET /api/products` | 只读加载职达在线产品目录 | 已接通 |
| `GET /api/system/status` | 检查岗位、职业情报、RAG、Dify 和会话保护状态 | 已接通 |
| `GET /api/intelligence/v1/jobs/search` | 同源代理职业情报岗位搜索 | 已接通，只读白名单 |
| `POST /api/intelligence/v1/decisions/evaluate` | 用官方证据核验目标岗位 | 已接通，最小字段且不转发身份凭据 |
| `POST /api/advisor` | RAG 检索后请求 Dify，并校验引用 | Mac mini 公开知识库测试链已在线；正式登录授权未接入 |
| `POST /api/zhida-connect/start` | 发起主站一次性资料授权 | Agent 侧已实现；未配置可信端点时关闭 |
| `POST /api/zhida-connect/complete` | 服务端兑换脱敏快照并签发加密会话 | Agent 侧已实现；正式主站兑换端点待部署 |
| `GET/DELETE /api/zhida-connect/session` | 读取脱敏档案/权益或断开 | 已实现，不保存主站长期 token |
| `GET/PUT/DELETE /api/workspace` | 读取、保存或删除当前匿名用户的目标岗位与任务进度 | 已实现；未连接或目录未配置时关闭，不接收个人档案 |

岗位与产品的默认只读数据源为 `https://www.zhidasihai.cn/api/trpc`。未连接主站接力时，产品接口只能确认在线目录，页面不会声称用户已拥有某项权益；只有主站签发的最小快照中 `allowed=true` 的功能才显示“直接使用”。

## 环境变量

示例见 `.env.example`。所有变量都只能配置在服务端，不能使用 `NEXT_PUBLIC_` 暴露到浏览器。

```bash
# 可选；不填时使用职达线上只读 tRPC 地址
ZHIDA_TRPC_URL=https://www.zhidasihai.cn/api/trpc

# 独立职业情报只读 API；只允许由本项目服务端访问
CAREER_INTELLIGENCE_API_URL=http://127.0.0.1:18080

# 必填后才启用知识库检索；密钥可按现有 RAG 接口要求决定是否填写
RAG_API_URL=http://localhost:8001/search
RAG_API_KEY=

# 两项都必须填写；可填 Dify /v1 基址或完整 /chat-messages 地址
DIFY_API_URL=http://localhost:8000/v1
DIFY_API_KEY=

# DeepSeek 直连；配置后优先于 Dify
DEEPSEEK_API_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=
DEEPSEEK_MODEL=deepseek-v4-flash
# 为节省 Token，默认关闭思考并把单次输出限制为 700 tokens
DEEPSEEK_THINKING_ENABLED=false
DEEPSEEK_MAX_OUTPUT_TOKENS=700

# 必填且至少 32 位，用于签名匿名 HttpOnly 会话
ADVISOR_SESSION_SECRET=

# 默认关闭；仅当知识库全部属于可公开资料时，才可显式开启匿名本地测试
ADVISOR_ALLOW_ANONYMOUS_PUBLIC_KB=false

# 主站资料接力；正式环境的两个端点必须同属一个 HTTPS origin
ZHIDA_AGENT_AUTHORIZE_URL=
ZHIDA_AGENT_EXCHANGE_URL=
ZHIDA_AGENT_SESSION_SECRET=
ZHIDA_AGENT_AUDIENCE=job-search-agent

# Agent 自有匿名状态目录；必须是服务端绝对路径
JOB_AGENT_WORKSPACE_DIR=/Users/your-name/.local/share/tokensoff/workspaces
```

AI 顾问默认不允许匿名检索。知识库含简历、求职案例或内部资料时，不要开启匿名测试开关，必须先接入真实登录和知识库访问授权。即使 RAG、AI 服务与会话密钥都已填写，访问许可未开启时接口仍会在检索前拒绝请求，不会把知识库片段返回浏览器。

RAG 接口约定为：

```http
POST RAG_API_URL
Content-Type: application/json
Authorization: Bearer <RAG_API_KEY>  # 配置密钥时才发送

{
  "query": "学生的问题",
  "topK": 6,
  "profile": {
    "degreeLevel": "bachelor",
    "major": "计算机科学与技术",
    "graduationYear": 2027
  },
  "target": {
    "companies": ["国家电网"],
    "jobTitles": ["信息技术"]
  },
  "filters": {
    "validAt": "2026-07-13",
    "status": "open"
  }
}
```

返回值可以使用 `results`、`records`、`items`、`data` 或 `chunks` 列表。每条有效资料至少要包含标题和正文片段，例如：

```json
{
  "results": [
    {
      "id": "notice-001",
      "title": "招聘公告标题",
      "snippet": "与问题相关的原文片段",
      "url": "https://example.com/notice",
      "publishedAt": "2026-07-01",
      "score": 0.91
    }
  ]
}
```

Dify 应用需要支持聊天消息的流式响应，并预先声明以下 6 个字符串输入变量：

- `policy_version`
- `system_policy`
- `profile_context`
- `target_context`
- `reference_date`
- `retrieval_context`

回答中的招聘事实必须使用 `[资料1]`、`[资料2]` 这类编号引用。没有有效引用、引用越界、Dify 流程失败或连接提前结束时，服务端会拒绝模型归纳并安全降级为官方原文。即使模型回答通过校验，服务端仍附上与问题最相关的官方原文核验区；比较多个招聘计划时分别附源。选定目标企业后，真实但属于其他企业或同集团兄弟企业的资料也会被拒绝；全部不匹配时返回 `NO_GROUNDED_EVIDENCE`。公告原文明确写出报名/投递截止日时，系统会结合 `reference_date` 标注“已截止”或“尚未到截止日”。

## 真实知识库与 Mac mini 部署

- 知识库说明和来源审核流程见 [`services/knowledge-base/README.md`](services/knowledge-base/README.md)。2026-07-18 的 Mac mini 运行快照为 17 个登记来源、12 个启用来源、81 条当前保存文档，其中动态质量分类为 61 条可进入回答的证据、20 条仅用于发现详情页的栏目索引、0 条残缺页；17 条当前正文由 Apple Vision 长图 OCR 补全，1 条中国石油公告由 Tesseract 补全，原始识别审计记录共 37 条，当前低质量 OCR 为 0。同一当前版本和图片 hash 会复用已审计 OCR，避免重复同步制造假版本；栏目遍历采用正文优先、分页渐进的受限队列，已知不支持的 Office/媒体附件不会被误报为网页抓取故障。国家电网、中国石油、中国石化、中国移动、中车长客和中国能建投资集团的六份高频企业官方公告已通过受限静态页面接入；招聘门户或历史公告的可访问性不被包装成实时在招。73 个 Dify 映射全部 `synced` 且正文 hash 一致，但检索只接受能映射到本地当前可回答正文并符合选定企业的片段。航天科技/航天科工以及中国石油/中国石化这类近名企业均做确定性隔离；多条件问题按招聘对象、学历、年龄、毕业时间、外语、专业、地点、单位、福利、流程和投递入口等分面，从本地当前正文补齐。服务端还会保留经过核验、位于页面版权信息之后的补充证据，避免页脚截断关键事实。同 URL 事实变化和跨 URL 公告变更都采用先隔离、后审核。教育部政策候选版经人工确认仅去除站点导航和页脚、政策正文完整后已批准并重新对账；审核队列现保留 3 条已停用旧入口的真实同步失败，不把超时、动态页面或 TLS 问题伪装成已解决。Mac mini 上一版知识库 98 项通过；本地新增部分岗位取消/核减、名额转移、反误判、专业要求、混合学历、分学历年龄/工作年限、英语等级分数、境内外毕业/认证日期、招聘对象、考试日期、精确报名截止、工作地点、招聘人数和投递入口门禁后为 189/189，尚未发布。规模仍不代表央国企岗位已经广泛覆盖，入口事实也不等于当前连通性验证，更不能据此生成无样本依据的录取概率。
- Mac mini 常驻、备份、Dify、Ollama 和 Cloudflare Tunnel 的运行说明见 [`infra/macmini/README.md`](infra/macmini/README.md)。Dify、知识库和 Ollama 只监听本机回环地址，公网只进入网页前端。
- 当前招聘岗位继续来自职达只读岗位接口；知识库负责政策、企业、要求和方法类依据。没有核验发布日期或有效期的旧资料，不能回答成“目前正在招聘”。

## 产品原则

- 事实、推断和建议分开呈现，事实必须附来源和更新时间。
- 不满足当前批次不等于“路径失败”：系统明确风险并保留长期目标，但不会把虚假路径包装成可投结果。
- 资料不足时明确标记未知或拒绝结论，不使用“稳进、保录、内部名额”等承诺。
- 产品只在对应任务卡点出现，始终标明可选，不把营销内容伪装成招聘结论。
- 本产品不是招聘单位官方平台，招聘条件以单位最新公告为准。
